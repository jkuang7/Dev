Recommendation: Vercel
- Best Next.js support, edge network covers America well
- Easy GitHub integration, automatic deploys
- Good-enough free tier to start
- Alternative: Railway if you want simpler pricing predictability
AWS is overkill and more complex to manage.
2. Database
You likely don't need one yet.
Your file-based approach is fine for 500 stocks:
- Store dcf_results.json in S3 or Vercel Blob
- CloudFront CDN in front for fast global access
- Cron job (GitHub Actions or external) runs nightly to update
Only add DB if you need:
- User accounts with personalized portfolios
- Real-time per-user filtering
- Tracking user preferences
Supabase (Postgres) would be $0-5/mo if you need it.
3. View Separation
Simple solution:
- /dcf - Public, read-only, shows cached results, no pipeline controls
- /admin - Protected route (password or auth), full pipeline access
Use NextAuth.js or simple password protection. Admin sees all tickers with run controls; public sees only pre-computed results.
4. User API Keys & Pricing
Hybrid model:
| User Type | Access | Cost |
|-----------|--------|------|
| Free visitors | Cached S&P 500 results | $0 |
| Paid members ($8/mo) | Full access to your pre-computed tickers | $8 |
| Power users | Analyze ANY ticker (uses YOUR API) | +$20/mo or pay-per-use |
Better approach for $8/mo:
- Users get access to your cached analyses
- For custom tickers: either require they provide their own OpenAI API key, or charge per-analysis credits
- Use OpenRouter to access multiple LLMs (some cheaper than GPT-4)
Key question for you: How important is it for free users to analyze ANY ticker vs only seeing your pre-computed S&P results?
